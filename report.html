练习：定义衡量标准¶

如果不能对模型的训练和测试的表现进行量化地评估，我们就很难衡量模型的好坏。通常我们会定义一些衡量标准，这些标准可以通过对某些误差或者拟合程度的计算来得到。在这个项目中，你将通过运算决定系数 R2 来量化模型的表现。模型的决定系数是回归分析中十分常用的统计信息，经常被当作衡量模型预测能力好坏的标准。

R2的数值范围从0至1，表示目标变量的预测值和实际值之间的相关程度平方的百分比。一个模型的R2 值为0还不如直接用平均值来预测效果好；而一个R2 值为1的模型则可以对目标变量进行完美的预测。从0至1之间的数值，则表示该模型中目标变量中有百分之多少能够用特征来解释。模型也可能出现负值的R2，这种情况下模型所做预测有时会比直接计算目标变量的平均值差很多。

在下方代码的 performance_metric 函数中，你要实现：

  • 使用 sklearn.metrics 中的 r2_score 来计算 y_true 和 y_predict的R2值，作为对其表现的评判。
  • 将他们的表现评分储存到score变量中。

问题 3- 训练及测试¶

将数据集按一定比例分为训练用的数据集和测试用的数据集对学习算法有什么好处？

提示： 如果没有数据来对模型进行测试，会出现什么问题？

答案:

学习算法的目的就是生成一个能够将数据集中的规律包含在内的模型，用以根据这个数据集的特征预测对应目标值，重点是能够预测并且预测准确率高。

如果将所有数据都拿去作为训练数据，那么即使模型能够对每一个特征组合都能给出准确的目标值，这个模型也不能保证对该数据集中尚未录入的数据点依然能做出正确的预测，预测的准确率也无法衡量。对已有数据推断出正确的目标值并不是我们最想解决的问题，事实上这样的模型有很多但都不是我们追求的结果，最简单的便是一一映射，对训练集内的数据可以进行准确无误的推断，但对我们感兴趣却不在训练集内的数据仍然毫无用处。

分为训练和测试的好处可以分为两点：

1.明确学习算法的目的。目的就是提高模型对未知数据预测的准确率，未知数据只不过是从所有已知数据集中抽出的一部分数据，模型无法感知这部分而已。

2.提供算法准确度衡量依据。模型在训练集中的准确度不能直接反应其预测能力，但是对测试集中的准确度将极大地反映出模型的预测能力，可以很好的用来评价模型的优劣。

问题 7- 网格搜索（Grid Search）¶

什么是网格搜索法？如何用它来优化学习算法？

回答: 在训练模型时通常我们会先选择一种对应的学习方法，一般情况下每个方法都会需要提前设定一系列参数，比如基函数的选择、决策树最大深度的选择，对于这些参数的选择，哪一种组合训练出的模型性能最好，事先是没法知道的。所以我们将这些参数分成不同的维度，将每一种参数的可能取值列出，组成一个参数组合网格，每一个单元就是一种参数组合，遍历每一种组合对模型进行训练，利用事先给定的评分标准筛选出认为性能最好模型所使用的参数，这种类似于在网格中通过遍历搜索最佳参数组合的方法就是网格搜索。由于网格搜索法对所有组合进行遍历，带来的计算量比较大，所以在一开始给出的参数组合要尽可能的合理，可以凭借经验也可以先小范围尝试，尤其是对于大数据量的训练集，通过这样的方式可以一定程度的优化学习算法。

问题 8- 交叉验证¶

什么是K折交叉验证法（k-fold cross-validation）？优化模型时，使用这种方法对网格搜索有什么好处？网格搜索是如何结合交叉验证来完成对最佳参数组合的选择的？

提示： 跟为何需要一组测试集的原因差不多，网格搜索时如果不使用交叉验证会有什么问题？GridSearchCV中的'cv_results'属性能告诉我们什么？

答案: 为了防止过拟合，训练模型前必须对数据集划分，其中一部分作为训练集另一部分作为测试集。而如何划分训练集与测试集影响到最终模型预测能力的真实水平。

K 折交叉验证法就是将训练集划分为 K 份，任取一份作为验证集，其余 K-1 份作为训练集，训练完成后记录下模型在该验证集的得分，由于每一份都可以作为验证集，所以经过 K 次训练后得到 K 个得分，再计算 K 次得分的平均值作为该模型的真实得分。该方法保证每次训练都保留一份数据用来验证，防止了训练可能出现的过拟合，而且由于训练集中所有数据都被验证过，也都被训练过，保证了最后得分的随机误差最小化。

网格搜索法的目的是找出一组参数，使得训练后的模型具有最佳的预测能力。当网格搜索法选定一组参数后，需要得到这组参数构成的模型的预测能力评分，该评分必须尽可能的反映其真实水平，而 K 折交叉验证法就可以保证该评分受随机因素影响最小并反映其真实预测水平。当网格搜索法穷举完所有参数组合，并得到每组参数组合的评分后，选择评分最高的一组便是预测能力最佳的一个模型了。如果不使用交叉验证法，则每组参数的得分都包含有一定随机误差，也就难以准确评估哪一组参数才是最佳模型。

‘cv_results’便是 GridSearchCV 最后输出的一个最佳结果，这是一个字典类型的对象，其中包含了最优参数组合的确切值，以及评分最高组的训练集和测试集，和具体的评分等内容，可以认为这就是该评分标准下的最佳模型。

练习：训练模型¶

在最后一个练习中，你将需要将所学到的内容整合，使用决策树演算法训练一个模型。为了保证你得出的是一个最优模型，你需要使用网格搜索法训练模型，以找到最佳的 'max_depth' 参数。你可以把'max_depth' 参数理解为决策树算法在做出预测前，允许其对数据提出问题的数量。决策树是监督学习算法中的一种。

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    cv_sets = ShuffleSplit(n_splits = 10, test_size = 0.20, random_state = 0)

首先观察数据集，将价格大于7450、建造时间早于1944、面积大于1124以及楼层大于91的数据点去除，因为这些数据点有可能属于异常值，这些异常值总共10个，相对于整个数据集9999个来说去除是可以接受的。

然后模型的训练方法选择为决策树，通过网格搜索法寻找最优模型参数，寻找的参照评分工具为模型的决定系数。将数据集中的千分之五作为测试集，大概有50个数据点，其余作为训练集。网格搜索法当中将训练集随机拆分为100份，每份大概100个数据点，将每份中的十分之一作为验证集。最大搜索深度设置为24。经过交叉验证以及对所有深度遍历，找到的最佳深度为23。

由于最后的决定系数得分不高，所以不太符合预期的效果。

分析原因可能有：

1、数据集中的房屋面积跨度从几十平米到一千多平米，房屋的性质从居民住房到厂房可能都有，而两种类型的房屋价格影响因素却没有考虑进来，这将使模型的泛化能力减弱。更好的做法应该是针对某一种类型的房屋进行建模并预测，这样的效果会更好。其他影响因素也有类似情况。

2、当影响因素较多时，决策树并不是最有力的学习训练方法，可以尝试其他方法进行学习训练以取得更好的模型。

交叉验证在网格搜索法寻找最佳参数时，能够保证每一组参数组合都有一个相对平均的评价，减少了因为参数组合相同、训练测试分组不同而导致的评价差异性，保证了搜索结果的可靠性，这对于提高模型最终的泛化能力非常重要。
